## 一、字典结构文档

### 1.1 数据库表结构总览

系统共有 **7 张字典相关表**，分为三层：

| 层级 | 表名 | 用途 | 数据量 |
|------|------|------|--------|
| 类目树 | `ozon_categories` | Ozon 三级类目树（L1/L2/L3） | 7,988 条 |
| 属性定义 | `ozon_category_attributes` | 每个 L3 类目的属性（特征）定义 | 258,614 条 |
| 字典值 | `ozon_attribute_dictionary_values` | 有 dictionary_id 的属性的可选值 | 按需同步 |
| 类目映射 | `category_mappings` | 采集源分类 → Ozon 类目的映射缓存 | 3 条 |
| 同步日志 | `ozon_category_sync_log` | 类目树/属性同步操作日志 | 6 条 |
| 同步进度 | `ozon_attr_sync_progress` | 批量属性同步断点续传 | 1 条 |

### 1.2 核心表字段说明

#### ozon_categories（类目树）

| 字段 | 类型 | 说明 |
|------|------|------|
| `description_category_id` | INTEGER | Ozon L1/L2 类目 ID |
| `type_id` | INTEGER | Ozon L3 类型 ID（叶子节点） |
| `parent_id` | INTEGER | 本表自增 ID，指向父节点 |
| `level` | INTEGER | 层级：1=L1, 2=L2, 3=L3 |
| `category_name` | TEXT | 合并名 "中文（俄语）" |
| `category_name_zh` | TEXT | 中文名 |
| `category_name_ru` | TEXT | 俄语名 |
| `disabled` | INTEGER | Ozon 是否已禁用该类目 |

#### ozon_category_attributes（类目属性定义）

| 字段 | 类型 | 说明 |
|------|------|------|
| `description_category_id` | INTEGER | 所属 L2 类目 ID |
| `type_id` | INTEGER | 所属 L3 类型 ID |
| `attribute_id` | INTEGER | Ozon 属性 ID |
| `name` | TEXT | 合并名 "中文（俄语）" |
| `name_zh` / `name_ru` | TEXT | 中/俄名 |
| `attr_type` | TEXT | Ozon 原始类型：String/Integer/Decimal/Boolean/URL |
| `is_required` | INTEGER | 是否必填 |
| `is_collection` | INTEGER | 是否多值（1=可多选） |
| `dictionary_id` | INTEGER | 字典 ID（>0 表示有预定义可选值） |
| `max_value_count` | INTEGER | 最大可选值数（0=不限） |
| `group_name` | TEXT | 属性分组名 |

**UNIQUE 约束**：`(description_category_id, type_id, attribute_id)` — 同一类目下同一属性唯一

#### ozon_attribute_dictionary_values（字典值）

| 字段 | 类型 | 说明 |
|------|------|------|
| `dictionary_id` | INTEGER | 所属字典 ID |
| `attribute_id` | INTEGER | 所属属性 ID |
| `description_category_id` | INTEGER | **类目上下文**（区分共享字典） |
| `type_id` | INTEGER | **类目上下文**（区分共享字典） |
| `value_id` | INTEGER | Ozon 字典值 ID |
| `value` | TEXT | 合并值 "中文（俄语）" |
| `value_zh` / `value_ru` | TEXT | 中/俄值 |

**UNIQUE 约束**：`(dictionary_id, description_category_id, type_id, value_id)` — 同一字典在同一类目下值唯一

> **关键设计**：`dictionary_id` 在 Ozon 中可能被数百个类目共享（如"类型"属性 dictionary_id=1960），不同类目返回不同可选值，因此必须用三元组 `(dictionary_id, description_category_id, type_id)` 定位。

### 1.3 命名规范

| 数据类型 | 命名格式 | 示例 |
|----------|----------|------|
| 类目名 | `中文名（俄语名）` | `玩具（Игрушки）` |
| 属性名 | `中文名（俄语名）` | `品牌（Бренд）` |

| 俄语为空时 | 仅中文名 | `直杆伞` |

### 1.4 属性类型映射规范

| Ozon attr_type | 前端 type | 渲染方式 | 说明 |
|----------------|-----------|----------|------|
| `String` + dictionary_id=0 | `text` | 文本输入框 | 自由输入字符串 |
| `String` + dictionary_id>0 | `select` | 下拉选择（单选） | 从字典值中选择 |
| `String` + dictionary_id>0 + is_collection=1 | `select` | 弹窗多选 | 从字典值中多选 |
| `Text` | `textarea` | 多行文本 | 长描述 |
| `Integer` | `number` | 数字输入 | 整数 |
| `Decimal` | `number` | 数字输入 | 小数 |
| `Boolean` | `boolean` | Yes/No 下拉 | 是/否 |
| `URL` | `url` | URL输入 | 链接 |

| 8 | `get_attributes_with_dict` 对 dictionary_id>0 强制 type=select | 代码修复 | 属性返回 | 2026-06-29 |

### 2.2 详细变更说明

#### 变更 1-3：Schema 迁移（共享字典污染修复）

**问题**：`dictionary_id=1960`（"类型"属性）被数百个类目共享，旧表仅按 `dictionary_id` 存储值，后同步的类目覆盖先前类目的值。伞类目的"类型"下拉显示"毛绒玩具"。

**修复**：
- 新增 `description_category_id` 和 `type_id` 列
- 所有查询/写入按三元组 `(dictionary_id, type_id, description_category_id)` 定位
- 模型方法 `find_by_dictionary_id`、`has_values`、`replace_for_dictionary` 均新增上下文参数

**涉及文件**：
- `backend/schema.sql` — 表定义
- `backend/db.py` — 迁移逻辑 `_run_migrations()`
- `backend/models/category.py` — `AttributeDictionaryValue` 类
- `backend/services/category_sync_service.py` — 同步服务
- `backend/routes/category_routes.py` — API 路由

#### 变更 4：旧数据清理

清理 110,384 条 `description_category_id=0` 的旧格式数据，全部为污染数据。

#### 变更 7-8：属性类型映射修复

**问题**：所有 99 个属性（包括 61 个有 dictionary_id 的）都返回 `type: 'text'`，导致下拉框不渲染。

**修复**：
- `get_attributes_with_dict`：`dictionary_id > 0` 时强制 `type='select'`
- `_map_attr_type`：`Boolean` → `'boolean'`（原误映射为 `'select'`），新增 `URL` → `'url'`

### 2.3 前端多选交互变更

| 变更项 | 说明 |
|--------|------|
| `is_collection` 属性渲染方式 | 从 `<select multiple>` 改为弹窗选择（显示标签+搜索+复选框） |
| 数据收集格式 | 从 `dictionary_value_id` 单值改为 `dictionary_value_ids` 数组 |
| 发布格式 | `values: [{ dictionary_value_id: v1 }, ...]` 支持多值 |
| 回填逻辑 | 支持从缓存/API 查找标签文本渲染 |

### 2.4 版本控制策略

| 维度 | 策略 |
|------|------|
| Schema 版本 | `db.py` → `_run_migrations()` 幂等迁移，`init_db()` 启动时自动执行 |
| 数据同步 | `ozon_category_sync_log` 记录每次同步类型/状态/耗时/错误 |
| TTL 策略 | 类目树 30 天、属性列表 6 小时、字典值 24 小时 |
| 回滚 | 迁移为增量式（ALTER TABLE ADD COLUMN），不删除旧列，可向前兼容 |

## 三、字典管理机制

### 3.1 数据分类标准

```
Ozon 字典数据
├── 类目树（ozon_categories）
│   ├── L1 一级类目（25 个）— description_category_id
│   ├── L2 二级类目（541 个）— description_category_id
│   └── L3 三级类型（7,422 个）— type_id（叶子节点，选择后拉取属性）
│
├── 属性定义（ozon_category_attributes）
│   ├── 按类目分组 — (description_category_id, type_id) → 属性列表
│   ├── 按 group_name 分组 — 同一类目内属性分组展示
│   ├── 按类型分类 — String(75%) / Decimal(14%) / Integer(7%) / URL(3%) / Boolean(0.5%)
│   └── 按字典分类 — 有字典(86,771) / 无字典(171,843)
│
├── 字典值（ozon_attribute_dictionary_values）
│   └── 按上下文隔离 — (dictionary_id, type_id, description_category_id) 三元组定位
│
└── 类目映射（category_mappings）
    ├── 自动映射 — 关键词匹配 + AI 匹配
    └── 手动映射 — 运营人工添加
```

### 3.2 数据更新流程

```
┌──────────────────────────────────────────────────────────┐
│                  字典数据更新流程                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  1. 类目树同步（30天TTL）                                  │
│     trigger: 启动时检查 / 手动触发                         │
│     source: Ozon API /v1/description-category/tree       │
│     target: ozon_categories（全量替换）                    │
│     log: ozon_category_sync_log                          │
│                                                          │
│  2. 属性定义同步（按需）                                   │
│     trigger: 用户选择 L3 类目时                            │
│     source: Ozon API /v1/description-category/attributes │
│     target: ozon_category_attributes（按 type_id 写入）   │
│     cache: 内存+文件双级缓存（6小时TTL）                    │
│                                                          │
│  3. 字典值同步（按需，异步）                               │
│     trigger: 用户打开下拉/弹窗时                           │
│     source: Ozon API /v1/description-category/attribute/ │
│             values（limit=5000，减少翻页）                 │
│     target: ozon_attribute_dictionary_values             │
│             （按三元组上下文存储）                          │
│     async: 后台线程同步，前端轮询（2秒间隔）                │
│                                                          │
│  4. 类目映射缓存                                          │
│     trigger: 采集商品时自动匹配                            │
│     source: 关键词匹配 → AI 匹配                          │
│     target: category_mappings（命中缓存，跳过打分）         │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 3.3 质量保障规则

| 规则 | 实现方式 |
|------|----------|
| 字典值不跨类目污染 | 三元组 `(dict_id, type_id, desc_cat_id)` 隔离 |
| 属性类型映射正确 | `dictionary_id > 0` 强制 `type=select` |
| 必填属性校验 | 前端 `validateAllRequiredAttrs` + 后端校验 |
| 同步幂等性 | `UNIQUE` 约束 + `INSERT OR REPLACE` |
| 同步断点续传 | `ozon_attr_sync_progress` 记录进度 |
| 卡死日志自愈 | 启动时检查 `status=running` 超时记录 |
